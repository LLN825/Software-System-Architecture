# Software-System-Architecture
Document my journey to becoming an architect

按道理讲，架构师和项目经理是两个不同的职位。架构师更偏向技术类的工作，项目经理更偏向管理类的工作。然而在IT领域，很多场景下架构师同时会出任项目经理，因为了解了软件项目的实现，才能更好地进行预算控制、时间进度管理等。

架构师不会收集和分析需求但不代表没有需求分析能力，相反地，架构师更需要对业务发展有准确认知，对相关技术充分理解，从而对工程项目的技术方案乃至技术体系进行评审和决断。

技术先深后广，架构师必须精通领域内的一些技术，因为只有在精通了这些技术之后，才能摸清系统的流程原理，以更高的视角看清系统的整体，此时才有能力和条件去探寻新的技术，拓宽自身领域的边界。如此循环往复，最终达到即广又深

实践经验和理论方法要并重。架构师不仅要具备在问题领域上的经验，也需要具备在工程领域内的经验。不仅能将业务问题转化为软件实现，还要有良好的抽象思维和系统分析能力。二者融汇贯通后形成技术上的远见和整合能力。

而当你成为了他，你会发现种种困境与无奈，但是不要着急，道路是曲折的，前途是光明的。一般的Java程序员到能做架构师一般需要五年以上的时间，可能是6、7年。但有一个前提，是能参与到架构项目实践中。也就是说，程序员很难指望通过加深当前工作的方式成为架构师，而是一定要通过架构师的面试（亦或是有相应的工作变动），然后在架构项目通过实践由弱变强，最终成为合格的架构师。

系统架构设计师的名头是对你过往成绩的一次认证，同时也是你继续出发的一个起点。如果你以系统架构设计师这个证书为终极目标，那么考与不考意义不大。如果你仅仅把系统架构设计师看成是一个里程碑，继续在此基础上努力奋斗学无止境，那么我提前恭喜你，相信总有一天你会是所有架构师的骄傲。
